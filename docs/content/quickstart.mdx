# Quickstart

Coyote is composed of a set of well integrated infrastructure primitives for backend and data engineers. For more information on each module, please refer to their respective documentation section.

The focus of this page is to help you get started with using the Coyote API.

## Modules

The main concept you need to know when using Coyote is `module`, which is how we call the types of infrastructure primitives in Coyote such as `cache`, `kv-store`, `rate-limiting`, and `stream`.

All modules support having multiple entities under them, each addressed with its own `key`. For example the cache module may have multiple items, the rate-limiter may have multiple different rate-limiters, and the queue module may have many different queues; each with their own key identifying the specific resource.

Coyote also support [`namespace`s](./namespaces.mdx), which is a way of having different configurations (such as persistence) for a group of keys within the same module. So you can, for example, have one cache store with a maximum size of 50 GB that is persisted to disk, and a different cache store with a maximum size of 1 GB that is ephemeral. If not set explicitly, the default namespace is used.

Example usage (setting a KV key):

<CodeTabGroup>
<CodeTab>

```js
import { Coyote } from "coyote";

const coyote = new Coyote("AUTH_TOKEN");
const app = await coyote.kv.set("key", "value");
```

</CodeTab>
<CodeTab>

```python
from coyote import Coyote

coyote = Coyote("AUTH_TOKEN")
app = coyote.kv.create("key", b"value")
```

</CodeTab>
<CodeTab>

```rust
use coyote::api::Coyote;

let coyote = Coyote::new("AUTH_TOKEN".to_string(), None);
coyote
    .kv()
    .set(
        "key",
        b"value",
        None,
    )
    .await?;
```

</CodeTab>
<CodeTab>

```go
import (
	coyote "github.com/coyote/coyote-webhooks"
)

coyoteClient := coyote.New("AUTH_TOKEN", nil)
app, err := coyoteClient.Kv.Set(ctx, "key", b"value", nil)
```

</CodeTab>
<CodeTab>

```java
import com.svix.Coyote;

Coyote coyote = new Coyote("AUTH_TOKEN");
coyote.getKv().set("key", b"value")
```

</CodeTab>
<CodeTab>

```kotlin
import com.svix.kotlin.Coyote

val coyote = Coyote("AUTH_TOKEN")
coyote.kv.set("key", b"value")
```

</CodeTab>
<CodeTab>

```ruby
require "coyote"

coyote = Coyote::Client.new("AUTH_TOKEN")
coyote.kv.set("key", b"value", null)
```

</CodeTab>
<CodeTab>

```csharp
var coyote = new CoyoteClient("AUTH_TOKEN");
await coyote.Kv.CreateAsync("key", "value");
```

</CodeTab>
<CodeTab>

```php
$coyote = new Coyote('AUTH_TOKEN');

$app = $coyote->kv->set("key", "value");
```

</CodeTab>
<CodeTab>

```shell
export COYOTE_AUTH_TOKEN='AUTH_TOKEN'
coyote kv set '{ "key": "key", "value": "value" }'
```

</CodeTab>
