# Quickstart

Diom is composed of a set of well integrated infrastructure primitives for backend and data engineers. For more information on each module, please refer to their respective documentation section.

The focus of this page is to help you get started with using the Diom API.

## Modules

The main concept you need to know when using Diom is `module`, which is how we call the types of infrastructure primitives in Diom such as `cache`, `kv-store`, `rate-limiting`, and `stream`.

All modules support having multiple entities under them, each addressed with its own `key`. For example the cache module may have multiple items, the rate-limiter may have multiple different rate-limiters, and the queue module may have many different queues; each with their own key identifying the specific resource.

Diom also support [`namespace`s](./namespaces.mdx), which is a way of having different configurations (such as persistence) for a group of keys within the same module. So you can, for example, have one cache store with a maximum size of 50 GB that is persisted to disk, and a different cache store with a maximum size of 1 GB that is ephemeral. If not set explicitly, the default namespace is used.


Example usage (setting a KV key):

<CodeTabGroup>
<CodeTab>

```js
import { Diom } from "diom";

const diom = new Diom("AUTH_TOKEN");
const app = await diom.kv.set("key", "value");
```

</CodeTab>
<CodeTab>

```python
from diom import Diom

diom = Diom("AUTH_TOKEN")
app = diom.kv.create("key", b"value")
```

</CodeTab>
<CodeTab>

```go
import (
	diom "github.com/diom/diom-webhooks"
)

diomClient := diom.New("AUTH_TOKEN", nil)
app, err := diomClient.Kv.Set(ctx, "key", b"value", nil)
```

</CodeTab>
<CodeTab>

```rust
use diom::api::Diom;

let diom = Diom::new("AUTH_TOKEN".to_string(), None);
diom
    .kv()
    .set(
        "key",
        b"value",
        None,
    )
    .await?;
```

</CodeTab>
<CodeTab>

```java
import com.svix.Diom;

Diom diom = new Diom("AUTH_TOKEN");
diom.getKv().set("key", b"value")
```

</CodeTab>
<CodeTab>

```kotlin
import com.svix.kotlin.Diom

val diom = Diom("AUTH_TOKEN")
diom.kv.set("key", b"value")
```

</CodeTab>
<CodeTab>

```ruby
require "diom"

diom = Diom::Client.new("AUTH_TOKEN")
diom.kv.set("key", b"value", null)
```

</CodeTab>
<CodeTab>

```csharp
var diom = new DiomClient("AUTH_TOKEN");
await diom.Kv.CreateAsync("key", "value");
```

</CodeTab>
<CodeTab>

```php
$diom = new Diom('AUTH_TOKEN');

$app = $diom->kv->set("key", "value");
```

</CodeTab>
<CodeTab>

```shell
export DIOM_AUTH_TOKEN='AUTH_TOKEN'
diom kv set '{ "key": "key", "value": "value" }'
```

</CodeTab>
