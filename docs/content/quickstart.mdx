## Core concepts



- `module`: a type of infrastructure primitive such as `cache`, `kv-store`, `rate-limiting`, `stream`, and more.
- `key`: the unique key each entity gets in each module. E.g. each cache item, rate-limiter, and queue will have their own key.
- `namespace`: a group of keys within each module that has its own configuration (such as persistence).

## API key

You can get an API key from the dashboard, or if self-hosting refer to the self-hosting page.

Example usage:

<CodeTabGroup>
<CodeTab>

```js
import { Coyote } from "coyote";

const coyote = new Coyote("AUTH_TOKEN");
const app = await coyote.kv.set("key", "value");
```

</CodeTab>
<CodeTab>

```python
from coyote import Coyote

coyote = Coyote("AUTH_TOKEN")
app = coyote.application.create("key", b"value")
```

</CodeTab>
<CodeTab>

```go
import (
	coyote "github.com/coyote/coyote-webhooks"
)

coyoteClient := coyote.New("AUTH_TOKEN", nil)
app, err := coyoteClient.Kv.Set(ctx, "key", b"value", nil)
```

</CodeTab>
<CodeTab>

```rust
use coyote::api::Coyote;

let coyote = Coyote::new("AUTH_TOKEN".to_string(), None);
coyote
    .kv()
    .set(
        "key",
        b"value",
        None,
    )
    .await?;
```

</CodeTab>
<CodeTab>

```java
import com.svix.Coyote;

Coyote coyote = new Coyote("AUTH_TOKEN");
coyote.getKv().set("key", b"value")
```

</CodeTab>
<CodeTab>

```kotlin
import com.svix.kotlin.Coyote

val coyote = Coyote("AUTH_TOKEN")

```

</CodeTab>
<CodeTab>

```ruby

coyote = Coyote::Client.new("AUTH_TOKEN")

```

</CodeTab>
<CodeTab>


</CodeTab>
