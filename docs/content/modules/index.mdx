# Modules

Modules are how Diom calls the types of infrastructure primitives in Diom such as cache, kv-store, rate-limiting, and message broker.

Diom modules are intended as powerful building blocks for developers to build with. Modules are designed to be ready-to-use, meaning that developers don't need to build custom scaffolding code to use them. Most common web development patterns are well known and understood, so Diom provides implementations for these common patterns instead of just low-level building blocks to build them with themselves.

The list of modules Diom supports is continuously evolving. So please let us know if there are any modules that you think Diom should include that it doesn't already.
