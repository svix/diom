# Key-Value Store (`kv`)

The `kv` module is Diom's key-value store module.

It's disk backed, and allows storing infinite keys with values up to a few MB in size (per value).

## Setting keys (`kv.set`)

When storing data in `kv` you need to pass the wanted key and value, and you can optionally pass additional configuration.

Some of the values you can pass to `kv.set`:
* `key`: a string with the key name.
* `value`: a byte array with the wanted value.
* `ttl`: optional expiry time, after which the key will be deleted.
* `version`: expected value version (see [OCC](#optimistic-concurrency-control-occ) section below)
* `behavior`: how to behave on conflicts. Allowed values:
    * `upsert` (default): update if key exists, insert if doesn't.
    * `insert`: fail if key exists, insert if doesn't.
    * `update`: update if key exists, fail if doesn't.

Example setting a value with a 60s TTL:

```rust
client.kv().set(
    "my-key".to_string(),
    b"value goes here".to_vec(),
    KvSetIn::new()
        .with_behavior(OperationBehavior::Insert)
        .with_ttl(Duration::from_secs(60)),
)
.await?;
```

## Getting keys (`kv.get`)

Returns `None` when a value is not found.


```rust
let out = client.kv()
    .get(
        "my-key".to_string(),
        KvGetIn::new()
    )
    .await?;

match out.value {
    Some(bytes) => println!("Got '{key}': {}", String::from_utf8_lossy(&bytes)),
    None => println!("Key '{key}' not found"),
}
println!("Version: {}", out.version);
```

## Deleting keys (`kv.delete`)

To delete a key, call the `kv.delete` operation with the wanted key:

```rust
client.kv().delete(
    "my-key".to_string(),
    KvDeleteIn::new()
)
.await?;
```


## Optimistic concurrency control (OCC)

Diom supports [optimistic concurrency control](https://en.wikipedia.org/wiki/Optimistic_concurrency_control) as a way to ensure data consistency. This is achieved using the `version` parameter that's supported by both `kv.set` and `kv.delete`.

The way it works is that when you fetch a key, you get a `version` identifier in addition to the `value`. You can then pass this `version` to subsequent calls to `kv.set` and `kv.delete` to ensure that the key hasn't been written to since you last read the value.

Example code:

```rust
let key = "test-key";

// Set the key. Pass 0 as the version to enforce the key isn't already set.
client.kv().set(
    key.clone(),
    b"value 1".to_vec(),
    KvSetIn::new()
        .with_version(0),
)
.await?;

// Get the value back
let out = client.kv()
    .get(
        key.clone(),
        KvGetIn::new()
    )
    .await?;

match out.value {
    Some(bytes) => println!("Got '{key}': {}", String::from_utf8_lossy(&bytes)),
    None => println!("Key '{key}' not found"),
}
println!("Version: {}", out.version);

// Conditional write using the version token
client.kv().set(
    key.clone(),
    b"updated value".to_vec(),
    KvSetIn::new()
        .with_version(out.version),
)
.await?;

// This write will fail, as version is stale.
client.kv().set(
    key.clone(),
    b"write will fail".to_vec(),
    KvSetIn::new()
        .with_version(out.version),
)
.await?;

// This delete will fail, as version is stale.
client.kv().delete(
    key.clone(),
    KvDeleteIn::new()
        .with_version(out.version),
)
.await?;
println!("Deleted '{key}'");
```
